# Contributing

Thank you for your interest in contributing. This project welcomes improvements, new templates, and bug fixes.

## How to Contribute

### Adding a New Template

1. Create a new directory under the appropriate category
2. Include a working `requirements.txt` or `package.json`
3. Add a minimal `README.md` explaining the architecture
4. Include a test or validation script
5. Open a pull request with a clear description

### Improving Existing Templates

- Fix bugs or broken dependencies
- Update outdated model references
- Add evaluation or benchmarking scripts
- Improve documentation

### Code Style

- Python: Follow PEP 8
- TypeScript: Use strict mode
- Keep dependencies minimal — only include what the template actually uses
- Add type hints where practical

## Review Process

All contributions are reviewed for:
- Working code (tested by the contributor)
- Clear documentation
- Minimal dependencies
- No external tracking or telemetry
